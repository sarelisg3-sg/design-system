## Decisiones técnicas de la sesión

### Textarea
- Idéntico a Input salvo el elemento (`<textarea>`) y `min-h-24 py-2 resize-y`
- Fix adicional: `eslint.config.mjs` — `storybook-static/**` añadido a `globalIgnores` (bug pre-existente)

### Card
- Props directas (no subcomponentes) — decisión explícita de Sareli para MVP
- `overflow-hidden` en el contenedor para respetar `border-radius` en header/footer
- Footer como `ReactNode` sin imponer flex layout

### Avatar
- `"use client"` — `useState` para `onError` de imagen rota
- `<img>` nativo (no `next/image`) — portabilidad para P08. ESLint rule deshabilitada a nivel de archivo con `/* eslint-disable @next/next/no-img-element */`
- Fallback: imagen → iniciales (`getInitials(name)`, max 2 chars) → ícono `User` de lucide-react

### Typography
- Variant-to-element map: no necesita prop `as`, el elemento se infiere del variant
- `cva` + `--ds-text-*` tokens. Leading/tracking usan las clases built-in de Tailwind (mismos valores que `--ds-leading-*`)
- `code` variant: inline (`<code>`), `font-mono bg-neutral-100 rounded px-1` — funciona anidado en `body`

### Modal
- `<dialog>` nativo + `showModal()` — cero dependencias extra, focus trap y Escape manejados por el browser
- `useId()` para IDs únicos de `aria-labelledby` / `aria-describedby`
- Backdrop click: `e.target === e.currentTarget` (dialog element directo, no hijos)
- Body scroll locked con `document.body.style.overflow = "hidden"` + cleanup en `useEffect`
- Panel: `flex flex-col max-h-[calc(90vh-2rem)]`, header/footer `shrink-0`, body `flex-1 overflow-y-auto`
- Stories con render functions + `useState` local (válido en Storybook 10 + Vite)
